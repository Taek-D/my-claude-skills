# Differentiation & Impact Quantification Guide

## 차별화 전략

### 핵심 원칙
포트폴리오 내 프로젝트 5-6개가 동일한 구조로 보이면 "템플릿 사용" 느낌. 각 프로젝트마다 아래 요소를 다르게:

### 1. 타입 믹스로 자연스러운 차별화

**이상적인 포트폴리오 구성 (5-8개 프로젝트)**:

| 타입 | 비중 | 역할 |
|------|------|------|
| Type 1 (Business) | 1-2개 | 비즈니스 임팩트 증명 |
| Type 2 (Analysis) | 1-2개 | 분석 역량 증명 |
| Type 3 (Product) | 1-2개 | 빌드 능력 + 프로덕트 센스 |
| Type 4 (Learning) | 0-1개 | 성장 마인드셋 |
| Type 5 (Automation) | 1-2개 | 실무 효율화 역량 |

타입이 다르면 템플릿 구조가 자연히 달라지므로 가장 효과적인 차별화 방법.

### 2. 강조점 다르게

**성과 중심** (Type 1):
```
## 📊 Performance Overview
• **+23% revenue** - A/B 테스트로 최적 가격 발견
```

**인사이트 중심** (Type 2):
```
## 📊 Key Findings
• **3개 유저 세그먼트** 발견 - 각각 다른 전략 필요
```

**프로덕트 중심** (Type 3):
```
## ✨ Feature Showcase
• **실시간 식물 성장** - Canvas API 60fps 렌더링
```

**효율 중심** (Type 5):
```
## ⚙️ Workflow Design
• **-100% 수작업** - 일 30분 가격 비교 완전 자동화
```

### 3. 섹션 순서 변형

| 전략 | 순서 | 적합한 프로젝트 |
|------|------|-----------------|
| 기본 (추천) | Overview → Problem → Solution → Takeaways | 대부분 |
| 성과 최우선 | Overview → Deployment → Impact → Takeaways | Type 1 운영 성과 |
| 학습 최우선 | Overview → Takeaways → Goal → Details | Type 4 커리어 전환 |
| 데모 최우선 | Overview → Feature → Architecture → Takeaways | Type 3 웹앱 |
| 자동화 최우선 | Overview → Workflow → Efficiency → Takeaways | Type 5 봇/도구 |

⚠️ 순서 변형은 특별한 이유가 있을 때만. 기본 순서가 F-Pattern 최적화됨.

### 4. 글로우 색상으로 시각 차별화

| 색상 | 의미 | 타입 매핑 |
|------|------|----------|
| teal | 데이터 분석 | Type 1, 2 |
| amber | 자동화 | Type 5 |
| red | 비즈니스 | Type 1 |
| purple | AI/ML | Type 1, 2, 4 |
| pink | 시각화/프로덕트 | Type 3 |

### 5. 차별화 체크리스트

- [ ] 다른 프로젝트와 **다른 타입**인가? (같은 타입이면 강조점이라도 다르게)
- [ ] 다른 **글로우 색상**을 사용하는가?
- [ ] **섹션 순서**에 변형이 있는가?
- [ ] **Extra-Label**이 고유한가? (같은 라벨 반복 금지)
- [ ] **시각적 요소** (표/다이어그램 스타일)가 다른가?

## 임팩트 정량화 공식

### 1. 시간 절약 → 비용 절감
```
절감 비용 = 절약 시간(h/주) × 인원 × 52주 × 시간당 인건비
```
예: 주 2.5시간 절약 × 5명 × 52주 × ₩30,000 = **연 ₩19,500,000**

### 2. 프로세스 효율 향상
```
효율 향상 = (개선 후 - 개선 전) / 개선 전 × 100%
```
예: 처리 시간 8시간 → 45분 = **-91% 개선**

### 3. 전환율/매출 개선
```
매출 증대 = 전환율 차이 × 월간 트래픽 × 평균 객단가
```
예: +0.8%p × 5,000명 × ₩80,000 = **월 ₩3,200,000**

### 4. 비용 절감
```
비용 절감 = 개선 전 비용 - 개선 후 비용
절감률 = 절감액 / 개선 전 비용 × 100%
```

### 5. 프로덕트 메트릭 (Type 3)
```
사용자 가치 = 기능 수 × 사용률 × 재방문율
기술 성과 = Lighthouse Score, FCP, 번들 사이즈
```

### 6. 자동화 효율 (Type 5)
```
자동화 ROI = 절약 시간 × 시간당 비용 / 개발 투자 시간
처리량 증가 = 자동화 후 처리량 / 수작업 처리량
```

## 정량화 팁

### 추정이 어려울 때
- **보수적 추정**: "최소한 주 2시간은 절약" (실제 5시간이어도)
- **범위 제시**: "월 ₩200만~₩500만 절감 추정"
- **단위당 가치**: "건당 처리 시간 8분 → 2분"

### 데이터 없을 때
- **업계 벤치마크**: "데이터 분석가 평균 시급 ₩30,000 기준"
- **가정 명시**: "월 방문자 5,000명 가정 시"
- **상대적 개선만**: "처리 시간 -60% 개선"

## 임팩트 서술

**✅ Good**:
- "API 응답 속도 **-84%** (8.2초 → 1.3초)"
- "데이터 정제 자동화로 팀 주 12시간 절약, 연간 **₩18M** 비용 절감"
- "Canvas 렌더링 CPU **3% 이하** (기존 15%에서 최적화)"
- "6개 사이트 가격 모니터링 **완전 자동화**, 월 15시간 절약"

**❌ Bad**:
- "성능이 많이 좋아졌어요" (정량화 없음)
- "비용을 절감했습니다" (얼마나?)
- "사용자들이 만족했습니다" (측정 지표 없음)
