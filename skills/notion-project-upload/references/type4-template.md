# Type 4: Learning Project Template

> Kaggle, 강의, 학습 프로젝트에 사용

## Template Structure

```markdown
> 💡 **"왜 배웠고, 무엇을 얻었는가"**
>
> 학습 목표 및 핵심 성과를 한 문장으로

---

## 🎯 Performance Overview

**30초 스캔용 - 학습 성과**

| 항목 | Before | After | Growth |
|------|--------|-------|--------|
| 기술 역량 | X/10 | Y/10 | **+Z%** |
| 실무 적용 | 0건 | N건 | **N건** |
| Competition | Top X% | Top Y% | **+Z%p** |

**Impact Summary**: 무엇을 학습했고 → 어떤 역량을 얻었으며 → 실무에 어떻게 적용했는지

---
```

## 권장 섹션 (프로젝트에 맞게 2-3개 선택)

### Option A: Learning Goal + Approach

```markdown
## 🎓 Learning Goal

**Why I Started**

실무에서 느낀 니즈, 현재 역량의 한계, 학습 동기를 커리어 목표와 연결해서 서술

**Why This Resource/Method** ← Approach 필드 반영

• 왜 이 강의/데이터셋/컴피티션을 선택했는가
• 다른 대안 검토 결과 (다른 강의, 다른 데이터셋 등)

**Goals**

| Goal | Target | Measurement |
|------|--------|-------------|
| 기술 역량 | X/10 | Self-assessment |
| Competition | Top X% | Kaggle LB |
| 실무 적용 | N개 프로젝트 | Portfolio |

**Learning Resources**: 강의/책/문서 목록 + 투자 시간
```

### Option B: What I Learned

```markdown
## 🛠️ What I Learned

**Concept 1: [개념명]**

• **What**: 핵심 개념 설명
• **Why Important**: 실무에서 왜 중요한지
• **My Practice**: 직접 실습한 내용

**Concept 2: [개념명]**

• **What**: 핵심 개념 설명
• **Why Important**: 중요한 이유
• **My Practice**: 실습 내용

(핵심 코드 10-15줄 — 가장 배운 점이 큰 부분만)
```

### Option C: Competition / Practice

```markdown
## 🏆 Competition Results

**Competition Overview**

• Task: Classification/Regression
• Dataset: X건, Y features
• Metric: AUC/RMSE/F1

**My Approach**

(Mermaid 다이어그램 — EDA→Feature Engineering→Model→Ensemble)

**Progress**

| Week | Focus | Score | Rank |
|------|-------|-------|------|
| 1 | Baseline | 0.XX | Top X% |
| 2 | Feature Engineering | 0.XX | Top X% |
| 3 | Tuning + Ensemble | 0.XX | **Top X%** |

**Final Result**: Top X% (Nth / Total teams)
```

### Option D: Before & After

```markdown
## 📈 Before & After

| 역량 | Before (0-10) | After (0-10) | Evidence |
|------|---------------|--------------|----------|
| 기술1 | X | Y | 근거 |
| 기술2 | X | Y | 근거 |
| 기술3 | X | Y | 근거 |

**Before**: 학습 전 상태 구체적으로
**After**: 학습 후 변화 구체적으로
```

### Option E: Real-world Application

```markdown
## 🔗 Real-world Application

**적용 사례 1: [프로젝트명]**

• Before: 기존 방법 + 성능
• After: 학습한 기법 적용 + 개선된 성능
• Impact: 비즈니스 효과

**적용 계획**: 향후 실무에 어떻게 활용할 것인지
```

### Option F: Community & Sharing

```markdown
## 🌐 Community & Sharing

• Kaggle Discussion/Blog: 공유한 내용 + 반응
• GitHub: 오픈소스 코드 + Stars
• 사내 세션: 공유 발표 + 참여자 수
```

## 필수 마무리 섹션

```markdown
---

## 💡 Reflection

**잘한 점**

• 체계적으로 실험한 부분, 효과적이었던 학습 방법

**아쉬운 점**

• 시도하지 못한 것, 다음에 개선할 점

**Next Step**: 다음 학습 계획

---

## 🔗 Links

[Kaggle Notebook](링크) | [GitHub](링크) | [Certificate](링크)
```

## 구성 예시

**프로젝트 A** (Kaggle 경진대회):
Overview → Learning Goal → Competition → Before & After → Reflection

**프로젝트 B** (기술 학습):
Overview → What I Learned → Real-world Application → Community → Reflection

**프로젝트 C** (자격증/코스):
Overview → Learning Goal → What I Learned → Before & After → Reflection

## Metric Scale Reference (Type 4)

| 항목 | 적절한 스케일 | 비현실적 |
|------|-------------|---------|
| Kaggle 순위 | Top 10-30% | Top 1% (최상위) |
| 학습 시간 | 40-120시간 | 500시간+ |
| 실습 프로젝트 | 2-5개 | 20개+ |
| 역량 변화 | 3→7 수준 | 0→10 수준 |

⚠️ Type 4는 가장 현실적인 수치를 유지하기 좋은 타입. 과장 없이 학습 과정과 성장을 보여주는 것이 핵심.
